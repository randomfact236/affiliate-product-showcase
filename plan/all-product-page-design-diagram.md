## ğŸ“ Implementation Notes

### Key Features

1. **Grid Layout**
   - Responsive columns: Desktop (4), Tablet (3), Mobile (1)
   - Gutter spacing: 20px between cards
   - Card aspect ratio: 3:4 (standard product image ratio)

2. **Filter System**
   - Multi-select categories, tags, ribbons
   - Active filters displayed as removable badges
   - Clear all filters resets to default view
   - Filter results update via AJAX (no page reload)

3. **Pagination**
   - URL-based pagination (products/page/2/)
   - Updates browser history on page change
   - Scrolls to top on page navigation
   - Disabled states for prev/next buttons

4. **Search**
   - Debounced input (300ms delay)
   - Real-time suggestions from user's search history
   - Pressing Enter or clicking search icon triggers search
   - Search terms highlighted in results

5. **Quick View Modal**
   - Opens on product card click or Quick View button
   - Shows larger product image and full details
   - Includes specifications, description, related products
   - Closes on X button or clicking outside modal

6. **Wishlist**
   - Heart icon toggle (empty/filled states)
   - Badge shows number of wishlist items
   - Tooltip: "Add to your wishlist"
   - Persists across sessions (localStorage or database)

7. **Pagination Per Page**
   - Options: 12, 24, 48, 96 products
   - Stored in user preferences
   - Default: 12 products per page
   - Updates URL on change

8. **Product Card Interactions**
   - Hover: Slight lift (4px up) + shadow increase
   - Image zoom: 1.1x scale transition
   - Buttons: Scale up on hover
   - Badge animations: Slide in on load

### API Integration

**REST Endpoints:**
- `GET /v1/products` - List all products
- `GET /v1/products?category={id}` - Filter by category
- `GET /v1/products?tag={id}` - Filter by tag
- `GET /v1/products?ribbon={id}` - Filter by ribbon
- `GET /v1/products?min_price={price}` - Filter by minimum price
- `GET /v1/products?max_price={price}` - Filter by maximum price
- `GET /v1/products?search={term}` - Search products
- `GET /v1/products?sort={field}` - Sort products
- `GET /v1/products?page={page}` - Pagination

**Query Parameters:**
- `per_page` - Items per page (12, 24, 48, 96)
- `page` - Page number (default: 1)
- `orderby` - Sort field (date, title, price, rating, menu_order)
- `order` - Sort direction (asc, desc)

**Response Format:**
```json
{
  "success": true,
  "data": [...products],
  "meta": {
    "total": 45,
    "per_page": 12,
    "current_page": 1,
    "total_pages": 4
  }
}
```
