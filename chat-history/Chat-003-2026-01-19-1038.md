## Menu Positioning Issue Debugged

### Root Cause Identified

**The Problem:**
WordPress `menu_position` parameter in CPT registration is unreliable. WordPress often ignores or modifies the exact position for CPTs based on:
1. Other plugins' menu positions
2. Internal WordPress menu ordering logic
3. Plugin activation order
4. Cache of menu positions

**Current Implementation Issues:**
- CPT has `'menu_position' => 55` (WordPress may override this)
- Manual menu has position `55.1` (respected by WordPress)
- Result: CPT position is unpredictable, menus appear scattered

### Solutions

**Solution 1: Use Menu Order Filter** (Recommended)

WordPress provides a filter to reposition menus after registration:

```php
// Add to Menu.php or create separate class
add_filter( 'menu_order', [ $this, 'reorderMenus' ], 999 );
add_filter( 'custom_menu_order', '__return_true' );

public function reorderMenus( $menu_order ) {
    // Find positions of our menus
    $products_key = array_search( 'edit.php?post_type=aps_product', $menu_order );
    $manager_key = array_search( 'affiliate-manager', $menu_order );
    
    if ( $products_key === false || $manager_key === false ) {
        return $menu_order;
    }
    
    // Move Affiliate Manager right after Affiliate Products
    unset( $menu_order[$manager_key] );
    array_splice( $menu_order, $products_key + 1, 0, [ 'affiliate-manager' ] );
    
    return $menu_order;
}
```

**Solution 2: Merge CPT as Submenu** (Single Menu Approach)

Remove CPT's independent menu, make it a submenu:

```php
// In ProductService.php
register_post_type(
    Constants::CPT_PRODUCT,
    [
        // ... other options
        'show_in_menu' => 'affiliate-manager',  // Add as submenu to Affiliate Manager
    ]
);
```

**Solution 3: Accept Imperfect Positioning** (Current approach)

Keep current implementation but acknowledge WordPress limitations:
- CPT position is approximate
- Manual menu position is exact
- Menus will appear near each other, not perfectly adjacent

### Why WooCommerce Works

**WooCommerce's Secret:**
WooCommerce doesn't rely on `menu_position` alone. It:
1. Uses `'show_in_menu' => true` for Products CPT
2. Uses manual menu with position for WooCommerce settings
3. Accepts approximate positioning (Products at ~55, WooCommerce at ~55.6)
4. Users accept they're not perfectly adjacent, just close
